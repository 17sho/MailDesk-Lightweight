# 安全政策

## 支持范围

安全修复优先面向 `main` 分支的最新代码。项目仍处于早期阶段，尚未承诺旧版本的长期安全维护。

## 私下报告漏洞

请使用 GitHub 仓库 **Security → Report a vulnerability** 创建私有安全报告。不要通过公开 Issue 披露可利用细节，也不要附带真实邮箱凭据、Token、Cookie、TOTP 密钥或私人邮件。

报告建议包含：

- 受影响的版本或提交；
- 漏洞类型和影响范围；
- 最小复现步骤或测试；
- 已脱敏的日志；
- 建议修复方式（如有）。

维护者会尽力在 7 天内确认收到报告，并在完成复现和风险评估后协调修复与披露时间。请勿对不属于你的邮箱、服务器或第三方服务进行测试。

## 安全边界

MailDesk 处理邮箱凭据、OAuth Token 和邮件正文，因此以下问题尤其重要：

- 明文凭据或秘密写入数据库、日志、导出或崩溃报告；
- DPAPI/Fernet 密钥管理或字段加密绕过；
- 导入、邮件 HTML、Webhook、代理配置导致的代码执行、SSRF 或路径穿越；
- 未确认的删除、移动、发件、转发等副作用；
- OAuth Token 泄露、证书验证关闭或重定向策略绕过；
- 打包产物加载非预期 DLL、Qt 插件或资源。
- 在线升级签名绕过、Release 清单混淆、暂存区 TOCTOU、事务锁失效或安装回滚失败；
- Ed25519 发布私钥疑似泄露。此类情况请立即私下报告，不要在公开 Issue 中张贴密钥材料。

在线升级的 Ed25519 私钥不保存在仓库、CI 配置或发行包中。客户端只内置公钥；密钥轮换需要发布安全公告，并为无法信任新根密钥的旧客户端提供一次手动升级路径。

纯功能缺陷、兼容性问题和不含敏感信息的错误日志可以通过普通 Issue 报告。
